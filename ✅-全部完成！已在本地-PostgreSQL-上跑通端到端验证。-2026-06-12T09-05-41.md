## 🎯 核心实现

### 1️⃣ QueryCondition 链式构造器

```python
from dal.condition import QC

# 等价 Java 原型
condition = (
    QC()
    .in_("saas_org_code", org_codes.split(","))
    .in_("saas_department_code", dept_codes.split(","))
)

# 支持的操作：eq / ne / in_ / nin / gt / gte / lt / lte / like / between / is_null / or_ / and_
complex_cond = QC().or_(
    QC().gt("age", 60),
    QC().eq("gender", "女"),
)
```

### 2️⃣ OperationDatabase 执行层

```python
from dal.operation import OperationDatabase as DB

# 查询列表（支持分页、排序、字段投影）
results = DB.query_list(
    Patient,
    condition,
    order_by=["age desc"],
    page=1,
    size=20,
    columns=["name", "age"],  # 可选：只返回指定字段
)

# 查询单条
patient = DB.query_one(Patient, QC().eq("id", 1))

# 计数
total = DB.count(Patient, QC().eq("status", 1))

# 新增（支持批量）
DB.insert([p1, p2, p3])

# 更新（返回影响行数）
affected = DB.update(Patient, QC().eq("id", 42), {"status": 0})

# 删除（返回影响行数）
deleted = DB.delete(Patient, QC().eq("status", 0))
```

## ⚠️ 上金仓后需重点回归测试的点

| 测试项 | 原因 | 验证方法 |
|--------|------|----------|
| **端口连接** | 金仓默认 54321，非 PG 的 5432 | 确认 `DAL_DATABASE_URL` 端口正确 |
| **标识符大小写** | 金仓 PG 兼容模式默认 lowercase，Oracle 模式默认 uppercase | 建表后查 `\d patient` 确认列名大小写 |
| **PG vs Oracle 兼容模式** | Oracle 模式下 SQL 语法差异大（如 `""` 引号规则） | 必须用 **PG 兼容模式**，否则需改方言 |
| **驱动兼容性** | 金仓自带 `ksycopg2`，可能与标准 `psycopg2` 行为略有差异 | 测试连接、事务、批量插入 |
| **`server_default=func.now()`** | 金仓对 `NOW()` 函数支持可能有差异 | 建表后插入记录，确认 `created_at` 自动填充 |
| **`IN` 子句性能** | 金仓对大列表 `IN` 的优化可能不同 | 测试 `in_` 传入 100+ 元素时的性能 |
| **`LIKE` 语义** | 金仓可能对中文 `LIKE` 有不同行为 | 测试 `like("name", "%张%")` |
| **事务隔离级别** | 金仓默认隔离级别可能与 PG 不同 | 测试并发更新场景 |
